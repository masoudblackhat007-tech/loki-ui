# بخش ۲۰ — اضافه کردن نمایشگر داخلی داکیومنت‌های دو‌زبانه

## تاریخ

```text
2026-05-11
```

## هدف

در این بخش برای پروژه `loki-ui` یک نمایشگر داخلی داکیومنت ساخته شد.

هدف این بود که مستندات پروژه فقط داخل فایل‌های Markdown repository باقی نمانند و از داخل خود UI هم قابل خواندن باشند.

این قابلیت برای داکیومنت‌سازی رزومه‌محور اضافه شد تا بخش‌های مختلف پروژه به شکل صفحه‌به‌صفحه، مرتب و قابل ارائه خوانده شوند.

## مشکل

قبل از این بخش، مستندات داخل مسیر `docs/progress` وجود داشتند، اما خواندن آن‌ها فقط از داخل فایل‌های repository ممکن بود.

همچنین پروژه به دو نسخه فارسی و انگلیسی نیاز داشت.

نسخه فارسی باید راست‌به‌چپ نمایش داده می‌شد و نسخه انگلیسی باید چپ‌به‌راست باقی می‌ماند.

اگر جهت متن درست تنظیم نمی‌شد، ترکیب متن فارسی با commandها، مسیر فایل‌ها، endpointها و نام سرویس‌ها خوانایی را خراب می‌کرد.

## تغییر انجام‌شده

یک route جدید برای خواندن داکیومنت‌ها اضافه شد:

```text
/docs
```

این route داخل همان برنامه `loki-ui` قرار گرفت.

کاربر می‌تواند با `Back` و `Next` بین بخش‌های داک جابه‌جا شود.

همچنین امکان انتخاب زبان فارسی یا انگلیسی اضافه شد.

## ساختار داک‌ها

داک‌ها به دو مسیر جدا منتقل شدند:

```text
docs/progress/fa
docs/progress/en
```

نسخه فارسی داخل مسیر `fa` قرار گرفت.

نسخه انگلیسی داخل مسیر `en` قرار گرفت.

نام فایل‌ها در هر دو زبان یکسان نگه داشته شد تا pagination بین دو زبان قابل پیش‌بینی باشد.

## پیاده‌سازی

در فایل `internal/httpserver/server.go` مسیر `/docs` ثبت شد.

در فایل `internal/httpserver/handler.go` منطق خواندن فایل‌های Markdown، انتخاب زبان، انتخاب صفحه و تعیین جهت متن اضافه شد.

در فایل `templates/docs.tmpl` صفحه خواندن داک ساخته شد.

## انتخاب زبان

زبان با query parameter زیر مشخص می‌شود:

```text
lang
```

برای فارسی:

```text
/docs?lang=fa&page=1
```

برای انگلیسی:

```text
/docs?lang=en&page=1
```

اگر زبان مشخص نشود یا مقدار نامعتبر باشد، زبان پیش‌فرض فارسی است.

## جهت متن

برای فارسی، مقدار `rtl` استفاده شد.

برای انگلیسی، مقدار `ltr` استفاده شد.

code blockها و inline codeها همیشه چپ‌به‌راست نمایش داده می‌شوند تا commandها، مسیرها و commit hashها خراب نشوند.

## تغییرات UI

صفحه داک یک کارت مرکزی دارد.

بالای صفحه زبان فعلی، شماره صفحه و نام فایل نمایش داده می‌شود.

دکمه‌های تغییر زبان در header قرار دارند.

دکمه برگشت به صفحه `Requests` حفظ شد.

پایین صفحه دکمه‌های `Back` و `Next` برای حرکت بین بخش‌ها قرار گرفتند.

داک‌ها در دو نسخه فارسی و انگلیسی قابل خواندن هستند.

نسخه فارسی راست‌به‌چپ نمایش داده می‌شود.

نسخه انگلیسی چپ‌به‌راست نمایش داده می‌شود.

## نکته امنیتی

این بخش مدل امنیتی را تغییر نداد.

دسترسی همچنان فقط از طریق SSH tunnel قابل قبول است.

پورت `18090` نباید public شود.

این بخش authentication، authorization، TLS، rate limiting یا audit logging اضافه نکرد.

## محدودیت این بخش

نمایشگر فعلی یک Markdown renderer ساده دارد.

این renderer برای فایل‌های کنترل‌شده داخل repository طراحی شده است.

این بخش search، table of contents یا deep link برای headingها اضافه نکرد.

## ارزش فنی

این بخش مستندات پروژه را از فایل‌های خام داخل repository به یک تجربه خواندنی داخل UI تبدیل کرد.

همچنین از ابتدا دو زبان و دو جهت متن را پشتیبانی کرد که برای داک فارسی و خروجی رزومه‌ای انگلیسی ضروری بود.
